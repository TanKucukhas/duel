## File Format Specifications

**Diablo 2 Data Formats (Custom Decoders):**
- DCC (DC6 format) - Character sprite frames with palette indices
  - Decoder: `tools/dcc_decoder.py` - BitReader-based binary parsing
  - Output: Individual frame pixel data with anchorX/anchorY metadata

- COF (Composite Object Format) - Animation sequence definitions
  - Specifies layer composition, frame counts, direction mappings
  - Used by `tools/generate_sprites.py` to assemble character animations

- DS1 (Map Layout) - Dungeon/town map tile grids
  - Decoder: `tools/ds1_decoder.py` - Parses wall, floor, shadow, object layers
  - References DT1 tiles by (orientation, mainIndex, subIndex)

- DT1 (Texture/Tile Data) - Tile graphics library
  - Decoder: `tools/dt1_decoder.py` - Block-based tile extraction
  - Isometric subtile layout (160x80 pixels per tile, 32x16 per subtile)

- PAL (Palette) - 256-color indexed color palettes
  - Format: 256 RGB triplets (768 bytes)
  - Separate palettes for each act (ACT1-ACT5)

**Generated Output Formats:**
- PNG - Spritesheets generated by `tools/generate_sprites.py`
- JSON - Atlas metadata: frame positions, dimensions, animation definitions
  - Examples: `demo/assets/sprites/ba/HTH/atlas.json`

## Data Pipeline

```
Diablo 2 ISOs/MPQs
  ↓
MPQExtractor (external tool - decompresses)
  ↓
Diablo2_all/extracted/ (DCC, COF, DS1, DT1, PAL files)
  ↓
Python decoders (tools/)
  ├→ dcc_decoder.py + generate_sprites.py → PNG spritesheets + JSON atlas
  ├→ ds1_decoder.py + dt1_decoder.py + render_map.py → PNG map image
  └→ Output: demo/assets/sprites/ and demo/assets/maps/
  ↓
PixiJS demo (client loads JSON atlas + PNG via HTTP)
  ↓
Browser rendering (WebGL via PixiJS)
```
